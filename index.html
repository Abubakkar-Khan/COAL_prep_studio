8086 & 80386DX Hardware Specs

Exam-focused processor specifications, pin groups, and signal uses. Learn what each group does, then redraw the matching diagrams from the Hardware filter.

808616-bit family1MB

8086 quick specification

  • Data bus16-bit external data bus, split into low/even and high/odd byte banks.
  • Address bus20 address lines, so maximum physical memory is 2^20 = 1MB.
  • Registers16-bit general registers, segment registers, IP, and FLAGS.
  • Memory modelReal-mode segment:offset addressing; physical address = segment * 10h + offset.
  • BanksA0 selects the low byte lane; BHE# enables the high byte lane.
  • ModesMinimum mode for single-processor systems; maximum mode uses external bus-controller support.
  • Exam lineWrite both widths: 16-bit data bus transfers data; 20-bit address bus selects up to 1MB memory.
80386DX32-bit family4GB

80386DX quick specification

  • Data bus32-bit external data bus, commonly drawn as four 8-bit byte lanes.
  • Address bus32 address lines, so maximum physical memory is 2^32 = 4GB.
  • Registers32-bit extended registers such as EAX, EBX, ECX, EDX, ESI, EDI, EBP, ESP, and EIP.
  • Memory modelSupports real mode and protected mode; architecture is ready for paging-based systems.
  • BanksBE0# to BE3# select which byte lanes D7-D0 through D31-D24 participate in a bus cycle.
  • ControlBus-cycle control signals identify address status, read/write direction, data/control cycle type, and ready completion.
  • Exam lineDraw four byte lanes and say byte-enable pins choose the active lane or lanes.

AD0-AD15I/OMultiplexed low address/data bus.Address appears first, then data uses the same pins.
A16/S3-A19/S6OutputHigh address lines multiplexed with status bits.Needed to complete the 20-bit address.
ALEOutputAddress Latch Enable.Latch AD lines during address phase before they become data.
BHE#Output, active lowBus High Enable for D15-D8.BHE# selects high/odd bank; A0 selects low/even bank.
RD#, WR#Output, active lowRead and write memory/I/O cycles.Active-low means 0 asserts the signal.
M/IO#OutputSelects memory cycle vs I/O cycle in minimum mode.In these notes: 0 selects I/O, 1 selects memory.
INTA#Output, active lowInterrupt acknowledge.Used after accepting a maskable INTR request.
INTR, NMIInputMaskable and non-maskable interrupt requests.NMI cannot be disabled by IF; INTR can be masked.
READYInputAllows slow memory/I/O to insert wait states.If not ready, CPU waits before completing the bus cycle.
HOLD, HLDAInput / outputDMA or another bus master requests and receives bus control.HOLD asks; HLDA grants.
LOCK#Output, active lowLocks the bus during critical bus sequences.Prevents other bus masters from taking the bus during locked operations.
TEST#Input, active lowUsed with WAIT-style synchronization.CPU can wait until TEST# becomes active.
QS0, QS1OutputInstruction queue status signals.Useful in maximum-mode style status explanations.
RESET, CLKInputReset initializes CPU; clock synchronizes operation.CLK drives timing; RESET starts execution from reset state.
MN/MX#InputSelects minimum or maximum mode.Minimum gives direct control pins; maximum uses status pins and controller.

80386DX pin groups and uses

Signal groupDirectionUseExam note
A2-A31OutputUpper address lines for the 32-bit physical address.Low byte position is handled by BE0#-BE3#, not A0/A1 in the same way.
D0-D31I/O32-bit data bus.Four byte lanes: D7-D0, D15-D8, D23-D16, D31-D24.
BE0#-BE3#Output, active lowByte-enable signals for the four data lanes.These decide which bytes are active in byte/word/dword transfers.
BS16#Input, active lowIndicates a 16-bit bus-size external transfer.Common exam trap: bus size support is separate from 32-bit address space.
ADS#Output, active lowAddress Status; valid bus-cycle address is available.ADS# does not mean address is over 20 bits; it marks a valid bus cycle.
W/R#OutputIdentifies write cycle vs read cycle.Combine with byte enables for per-bank write control.
D/C#OutputDistinguishes data cycle from control cycle.Useful in bus-cycle classification diagrams.
M/IO#OutputSelects memory access or I/O access.Do not confuse this with data-bus width.
READY#Input, active lowCompletes cycle when external system is ready.Inactive READY# inserts wait states.
INTR, NMIInputMaskable and non-maskable interrupt requests.Same concept as 8086: NMI has priority and is not maskable by IF.
HOLD, HLDAInput / outputBus arbitration with external bus masters.Used for DMA-style ownership transfer.
RESET, CLK2InputReset and clock timing.Clock controls bus timing; reset returns CPU to known state.
Exam answer pattern: first state the bus width, then state the addressable memory, then name the control pins that make the bus cycle legal. For diagrams, label active-low pins with # and explain what happens when the signal is 0.
